# Lab 14.5 – Real-Time Decision Making

## Objective

The objective of this laboratory is to convert the predictions generated by the trained deep learning model into executable control commands for a Brain–Computer Interface (BCI) system.

A confidence threshold is applied before issuing any command to ensure that only reliable predictions are translated into robot control actions.

This stage represents the decision-making layer of the real-time BCI framework.

---

## Background

Deep learning classifiers produce both a predicted class and an associated confidence value.

Although a prediction may be correct, executing commands with low confidence may introduce unwanted robot movements.

To improve safety and reliability, the system evaluates the confidence score before generating a command.

Predictions with confidence values above the predefined threshold are accepted, while uncertain predictions are rejected as **NO ACTION**.

This strategy is commonly adopted in real-time Brain–Computer Interface systems and robotic control applications.

## Processing Pipeline

1. Load prediction results.
2. Read prediction confidence values.
3. Define confidence threshold.
4. Convert prediction classes into control commands.
5. Reject uncertain predictions.
6. Save generated commands.
7. Generate command report.

---

## Decision Rules

| Prediction | Command |
|------------|---------|
| 0 | STOP |
| 1 | LEFT |
| 2 | RIGHT |

---

## Confidence Threshold

```
0.75
```

Predictions below this threshold become:

```
NO ACTION
```
