# User Guide — Hybrid Adaptive BCI Robotic Hand System (v1.0)

This guide covers installation, building, running, testing and troubleshooting for
both halves of the project: the **AI pipeline** (Python, runs anywhere) and the
**ROS2 robotics stack** (ROS2 Jazzy, runs on Linux / RunPod).

---

## 1. Dependencies

### AI pipeline (`requirements.txt`)

| Package | Purpose |
|---------|---------|
| mne, pooch | EEG loading, filtering, ICA, epoching, CSP |
| numpy, scipy, pandas | numerics / data |
| scikit-learn, joblib, xgboost | classical ML |
| tensorflow | CNN / LSTM / CNN-LSTM |
| matplotlib | figures |
| pytest, pytest-cov | tests |

Target interpreter: **Python 3.12**.

### ROS2 stack

| Component | Version |
|-----------|---------|
| ROS2 | Jazzy Jalisco (Ubuntu 24.04) |
| Simulator | Gazebo Harmonic (`ros-gz`) |
| Control | `ros2_control`, `ros2_controllers`, `gz_ros2_control` |
| Planning | MoveIt2 |

ROS2 build tooling is listed in `requirements-ros2.txt` for reference but is provided
by the ROS2 distribution / rosdep, **not** installed with pip.

## 2. Installation

### 2a. AI pipeline

```bash
git clone https://github.com/ROBOTICAbdelsalam/Hybrid-Adaptive-BCI.git
cd Hybrid-Adaptive-BCI
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2b. ROS2 stack (Linux / RunPod Ubuntu 24.04)

Option A — provided Docker image (reproducible):

```bash
cd ros2_workspace
docker build -t bci-ros2:jazzy -f docker/Dockerfile .
docker run -it --rm bci-ros2:jazzy
```

Option B — bootstrap a bare Ubuntu 24.04 pod:

```bash
bash ros2_workspace/docker/setup_runpod.sh
```

## 3. Build

### AI pipeline
No build step — the labs are Python scripts. Run them in order (see §5).

### ROS2 workspace

```bash
source /opt/ros/jazzy/setup.bash
cd ros2_workspace
bash docker/build_and_verify.sh      # colcon build + interface resolution check
# or manually:
colcon build --symlink-install
source install/setup.bash
```

## 4. Testing

### AI pipeline
Reports and figures under `results/` and `figures/` are regenerated by re-running the
labs. Key checkpoints (accuracies) are recorded in the reports.

### ROS2 packages (no ROS install required for the pure-logic tests)

```bash
# from a plain Python venv (33 tests):
cd ros2_workspace/src
for p in bci_robot_abstraction bci_bridge bci_hand_description \
         bci_hand_moveit_config bci_bringup; do
  (cd $p && PYTHONPATH=. pytest test/ -q)
done
```

Under ROS2 the same tests run via colcon:

```bash
cd ros2_workspace
colcon test
colcon test-result --verbose
```

**Test inventory (33):** abstraction 11 · bridge 8 · hand_description 6 ·
moveit_config 3 · bringup 5.

## 5. Run — AI pipeline

The labs run in numeric order. The real-time chain that feeds ROS2 is:

```bash
source .venv/bin/activate
python labs/lab14_01_live_eeg_streaming.py         # simulated EEG stream
python labs/lab14_02_real_time_preprocessing.py    # band-pass
python labs/lab14_03_online_csp_feature_extraction.py
python labs/lab14_04_real_time_prediction.py       # → realtime/results/realtime_predictions.csv
python labs/lab14_05_command_generation.py         # → realtime/results/realtime_commands.csv
python labs/lab13_02_adaptive_threshold.py         # → adaptive_ai/adaptive_threshold.csv
```

`realtime_predictions.csv` and `adaptive_threshold.csv` are exactly what the ROS2
bridge consumes.

## 6. Run — ROS2 execution

### Full system (EEG predictions → robotic hand)

```bash
source /opt/ros/jazzy/setup.bash
source install/setup.bash
ros2 launch bci_bringup bci_system.launch.py
```

This starts Gazebo with the hand, the robot node, the bridge and the hand adapter.
By default it replays bundled demo predictions so the hand visibly cycles through
gestures. To replay the real Lab 14 output instead:

```bash
ros2 launch bci_bringup bci_system.launch.py \
    predictions_csv:=$PWD/../realtime/results/realtime_predictions.csv \
    threshold_csv:=$PWD/../adaptive_ai/adaptive_threshold.csv
```

### Individual nodes (debugging)

```bash
ros2 run bci_robot_abstraction robot_node --ros-args -p robot_type:=robotic_hand
ros2 run bci_bridge bridge_node --ros-args \
     -p predictions_csv:=<path> -p threshold_csv:=<path>
ros2 run bci_bringup hand_trajectory_adapter
ros2 topic echo /bci/command
ros2 topic echo /bci/robot_state
```

## 7. Gazebo execution

```bash
# hand alone in Gazebo Harmonic with controllers:
ros2 launch bci_hand_description gazebo.launch.py

# confirm controllers are active:
ros2 control list_controllers
#   joint_state_broadcaster  active
#   hand_controller          active

# command a gesture directly (all fingers closed):
ros2 topic pub -1 /hand_controller/joint_trajectory trajectory_msgs/msg/JointTrajectory \
  '{joint_names: [thumb_joint,index_joint,middle_joint,ring_joint,little_joint],
    points: [{positions: [1.4,1.4,1.4,1.4,1.4], time_from_start: {sec: 1}}]}'
```

## 8. MoveIt2 execution

```bash
# planning demo with RViz:
ros2 launch bci_hand_moveit_config demo.launch.py
```

In RViz: set the planning group to **hand**, choose a named state
(`open_hand`, `close_hand`, `pinch_grip`, `power_grip`, `point`, `rest`) as the goal,
then **Plan** and **Execute**. `move_group` alone (no RViz) is available via
`ros2 launch bci_hand_moveit_config move_group.launch.py`.

## 9. Troubleshooting

| Symptom | Likely cause | Fix |
|---------|--------------|-----|
| `ModuleNotFoundError: mne` | AI deps not installed / wrong interpreter | `source .venv/bin/activate; pip install -r requirements.txt` |
| `pip install` fails on a wheel path | using an old `requirements.txt` | v1.0 `requirements.txt` has no local paths; re-pull |
| `colcon build` can't find `bci_interfaces` | workspace not sourced | `source install/setup.bash` after first build |
| Nodes can't import `bci_interfaces.msg` | interfaces package not built first | `colcon build --packages-select bci_interfaces` then the rest |
| Hand doesn't move in Gazebo | predictions below the confidence gate | lower `threshold_csv` value or use the demo data (default) |
| `hand_controller` inactive | spawner ran before the hand spawned | relaunch; the launch sequences spawners via event handlers |
| Gazebo GUI is black over SSH/RunPod | no display forwarding | run headless (`gui:=false`) or use X11/VNC; controllers still work |
| MoveIt plans but won't execute | controller name mismatch | `moveit_controllers.yaml` must name `hand_controller` (it does) |
| `xacro` error about `controllers_config` | arg not passed | the launch passes it; when running xacro by hand add `controllers_config:=<path>` |

For the pure-logic tests you need **no** ROS2 install — run them in the Python venv
(§4) to isolate whether an issue is in the logic or the ROS transport.
